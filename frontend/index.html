⏱ ASR Latencia
Dashboard en < 3 segundos
Con ~5,000 usuarios concurrentes, el dashboard de reportes debe responder en máximo 3s mediante caché Redis y agregación SQL.
—
Carga el dashboard para medir la latencia
Tácticas aplicadas
✦ Caché Redis — TTL 30s para dashboard, 60s para reportes
✦ Agregación SQL — GROUP BY en DB, sin Python-side loops
✦ Índices de DB — company_id, project_id indexados
✦ Pool de conexiones — SQLAlchemy connection pool
⚡ ASR Escalabilidad
100% de éxito en extracciones
El extractor debe capturar métricas de forma agnóstica en ambientes sobrecargados, garantizando 100% de éxito mediante reintentos.
—
Ejecuta una extracción para ver métricas
Tácticas aplicadas
✦ Reintentos (MAX 5) — Backoff exponencial 2^n segundos
✦ Jitter aleatorio — Evita thundering herd en reintentos
✦ Patrón Strategy — Provider agnóstico (AWS / GCP)
✦ Celery async — Cola de tareas para extracciones pesadas
✦ Validación — Solo métricas válidas pasan al resultado
